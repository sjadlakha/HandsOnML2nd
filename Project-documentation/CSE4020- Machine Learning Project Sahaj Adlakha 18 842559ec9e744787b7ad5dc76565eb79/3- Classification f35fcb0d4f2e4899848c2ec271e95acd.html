Training a Binary classifier

The 5 -detector

Stochastic Gradient Descent:

  • good for large dataset
  • Deals with each instance independently (one at a time, therefore well suited for online learning)
  • Relies on randomness during training

Metrics for SGD:

  1. Cross validation

    → Accuracy above 95%!!! (fishy)

2. Confusion Matrix

→A perfect classifier would have only true positives and true negatives (non zero values only on non diagonal elements)

3. Precision and Recall

→Even precise metric

→ F1 score = H.M of precision and recall (H.M. because it gives much more weights to low values)

Precision/ Recall Trade-off
  1. In applications, we need to consider precision (Restricted videos on Youtube) ⇒ less F.P.
  1. In other cases we need to consider recall (Detecting disease in a patient) ⇒ less F.N.
  • You can’t have both precision and recall high. If you increase precision, it will reduce recall, and vice versa.

⇒ "Generally" if you increase the threshold, precision will increase

4. Receiver Operating Characteristic (ROC) curve

→ True positive rate vs False positive rate

→ Sensitivity (recall) vs. (1- specificity)

Measure the area under the curve( ROC AUC )

2 Approaches:

  • OvR - One vs Rest : One Binary classifier for each class {preferred mostly}
  • OvO - One vs One: One binary classifier for each pair [ (n*(n-1))/2 ] { preferred in algos that do not scale with large data like SVM }
