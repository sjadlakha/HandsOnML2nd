Types of Machine Learning Systems

1. Supervised/ Unsupervised Learning

Supervised:

  • Training set includes desired solutions called labels.
  • kNN
  • Linear Regression
  • Logistic Regression
  • SVMs
  • Decision Trees and Random Forests
  • Neural Networks

Unsupervised Learning:

  • Training Data is unlabeled
  • Clustering
  • Anomaly Detection
  • Visualisation and dimensionality reduction
  • Association Rules

Semisupervised Learning:

  • Training data is partially labeled
  • Application:
    • Photo-hosting service like Google Photos
  • Combination of Supervised and Unsupervised Algorithms
    • Deep Belief Networks

Reinforcement Learning:

  • Learning System a.ka. AGENT: observes the environment, select and perform actions
  • Gets REWARDS in return .
  • It must then learn itself what is the best strategy , called a POLICY.

2. Batch and Online Learning

Based on whether or not the system can learn incrementally from a stream of incoming data.

Batch Learning:

  • Incapable of learning incrementally
  • Train the model with the entire data then push to production environment.
  • Any change in the training data will force you to repeat this step (but it can be automated easily)
  • - takes a lot of time and computing resources
  • Offline Learning

To adapt to changing data we need a more reactive solution üëáüèª

Online Learning:

  • Model is fed the data sequentially, either individually or in small groups called mini-batches.
  • + Each learning step is fast and cheap.
  • + It can learn on the fly
  • - If bad data is fed into the algorithm then the performance will decrease gradually in production environment.

3. Instance-Based vs. Model-Based Learning

  • Based on how the algorithms generalize.

Instance Based Learning:

  • Learn the examples by heart
  • Generalize on new instances based on a similarity measure

Model Based Learning:

  • Build a model based on the examples
  • Use this model to predict new-instances.

‚Üí Compare models based on the cost function

Main Challenges of Machine Learning

1. Insufficient Quantity of Training Data

  • Machine Learning requires a lot of data for most of the algorithms to train.
  • Even a simple problem requires thousands of examples.
  • Complex Problems like image recognition requires millions of examples.

2. Non-representative Training Data

  • Crucial to use a training set that is representative of the cases you want to generalize to.
  • Too small sample ‚áí Sampling noise(non-representative data as a result of chance)
  • But even large samples can be non-representative if sampling method is flawed‚áí Sampling bias

3. Poor-Quality Data

  • Outliers
  • Errors
  • Noise
  • Solution: Pre-processing

4. Irrelevant Features

Garbage in, Garbage out

Feature Engineering

  • feature selection
  • feature extraction
  • creating new features by gathering new data

5. Overfitting the Training Data

  • Model performs well on training data but does not generalize well.
  • Happens when the model is too complex relative to the amount and noise of the training data.

6. Underfitting the Training Data

  • Model is too simple to understand the underlying structure of the data
