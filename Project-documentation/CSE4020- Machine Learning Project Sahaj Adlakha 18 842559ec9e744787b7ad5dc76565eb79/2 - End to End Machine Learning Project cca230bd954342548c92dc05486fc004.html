2 - End to End Machine Learning Project

Steps:
1. Look at the big picture
2. Get the data
3. Discover and visualize the data to gain insights
4. Prepare the data for ML algorithms
5. Select a model to train it {Notebook}
6. Fine-tune your model.
7. Present you solution
8. Launch, monitor and maintain your system.

Steps:

1. Look at the big picture

→Frame the problem:

  • Is it supervised / unsupervised/ reinforcement?
  • Is it a classification task/ regression task
  • Should you use online-learning or batch learning

→Select a Performance Measure:

  • RMSE
  • MAE - when there are many outliers
  • Distance measures (lk norm)

→ Check the assumptions

2. Get the data

→ Create the workspace

  • Preferably a virtual and isolated env

→ Download the data

→ Take a look at the data structure

→ Train test Split {Notebook}( stratified sampling )

3. Discover and visualize the data to gain insights

→ Create visualisations

→ Look for correlations

→ Experiment with attribute combinations

4. Prepare the data for ML algorithms

→ Data cleaning

  • Missing data {Notebook - SimpleImputer}

→ Handling text and categorical data

  • ordinalencoder
  • onehotencoder
  • Custom transformers {utils.py}

→ Feature Scaling

  • min-max
  • standardization

→ Transformation Pipelines {Notebook}

  • Helps in preserving the order of multiple tasks that need to be performed

5. Select a model to train it {Notebook}

→ Training and Evaluating on the Training set

  • Try various models and compare their performance

→ Cross Validation {Notebook}

  • Great way to detect overfitting

→ Save a model (joblib/pickle)

6. Fine-tune your model.

→ Hyperparameter tuning:

  • Grid Search - get the best features, feature importance and much more
  • Randomized Search
    • For a larger hyperparameter space
    • In every iteration a random value is chosen for every hyperparameter and then the best ones are picked
  • Ensemble Methods
    • Combining the models that performed best
    • eg. Random forests

7. Present you solution

  • Before presenting test your system on the test set
  • Document everything with
    • Clear Visualisations
    • Easy-to-remember Statements

8. Launch, monitor and maintain your system.

  • Polish the code
  • Write Documentation
  • Write tests
  • Reuse the model
    1. Save and load it in another project
    1. Wrap the model in a web service that an application can query through REST API
    1. Deploy the model on cloud and make use of the cloud service provider API
  • Don't let the models "rot"
    • Take human help to detect any bugs or performance degradation
    • Automate tasks like collecting and loading fresh data
    • Keep backups of all models
